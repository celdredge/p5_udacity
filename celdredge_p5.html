Background¶

Summarize for us the goal of this project and how machine learning is useful in trying to accomplish it. As part of your answer, give some background on the dataset and how it can be used to answer the project question. Were there any outliers in the data when you got it, and how did you handle those? [relevant rubric items: “data exploration”, “outlier investigation”]

The goal of this project is to use machine learning to identify likely persons of interest (POI) from the Enron corporate fraud. The analysis uses a set of emails sent to and from Enron employees along with financial information related to employee compensation. Machine learning can assist in identifying patterns in email communication and employee compensation that predict POI likely to be involved in the fraud.

The dataset includes records for 146 Enron employees across 22 features. Only 18 of the 146 employees are labeled POI. The ratio of POIs to non-POIs in the dataset makes this project particularly challenging for machine learning. With only 18 POIs out of 146 individuals, the class imbalance means that special attention needs to be paid to classification, particularly during model evaluation and validation.

Several of the features in the dataset have missing values. In particular, the following features have many missing values: deferral_payments, deferred_income, director_fees, loan_advances, and restricted_stock_deferred. Additionally, the dataset includes a “TOTAL” record that is an outlier, likely a quirk of spreadsheet formatting that should be excluded. There are also two other records identified as outliers that were excluded due to missing data and likely not being an actual Enron employee ("LOCKHART EUGENE E" and "THE TRAVEL AGENCY IN THE PARK"). See notes and code below for more detail.

As part of the pipeline for each classifier, different values for k in the automated feature selection function SelectKBest were tested and performance was compared. The top performing classifier, the DecisionTreeClassifier, used the top 19 features identified by SelectKBest (see below for scores). Note that the scores discussed below are only intended to demonstrate the SelectKBest function. The actual feature selection takes place as part of the grid search for tuning the parameters of each classifier.

Included among the top features are two engineered features. The first is, "ratio_to_poi_to_all_sent," and is defined as the number of messages the person sent to a POI divided by the total number of messages sent by that person. The second is, "ratio_from_poi_to_all_received," and is defined as the number of messages received from a POI divided by the total number of messages recieved by that person. The intent of these additional features is to provide additional detail on how the person communicated with POIs, regardless of how often the person uses email. Even if an employee sends only a few emails to a POI, if that is a high proportion of the total emails that person sent, it may be meaningful.

When evaluating classifiers, I used the pipeline and grid search features of scikit-learn to scale features using the MinMaxScaler, select features, and classifier parameters. I generally tried a range of values for each parameter, but did not every parameter due to long run times. I also investigated feature importances from the decision tree classifier.

from sklearn.feature_selection import SelectKBest

def k_best(data_dict, feature_list, k):
    ''' sklearn SelectKBest feature selection, returns dict
    '''
    data = featureFormat(data_dict, feature_list)
    labels, features = targetFeatureSplit(data)
    
    kn = k + 1
    k_best = SelectKBest(k=(kn))
    k_best.fit(features, labels)
    scores = k_best.scores_

    unsorted_scores = zip(feature_list[1:], scores)
    sorted_scores = sorted(unsorted_scores, key=lambda x: x[1], reverse=True)
    k_best_features = dict(sorted_scores[:kn])
    df_k_best = pd.DataFrame(k_best_features, index=['score'])
    df_k_best = df_k_best.transpose()
    df_k_best = df_k_best.sort_values(['score'],ascending=False)
    print df_k_best
    return k_best_features

[relevant rubric item: “tune the algorithm”]

Tuning the parameters means adjusting the parameter arguments for the dataset and use case. If you don't do this well, the classifier can have poor performance during cross validation and fail to predict any unseen data. Cross-validation is used to tune the parameters of the classifiers. If we tuned parameters only using training data, the model would overfit. I used the grid search feature in scikit-learn to comprehensively compare a variety of parameter settings.

For example, for the Decision Tree classifier that ended up with the best performance, I varied the following attributes to see which yielded the highest F1 score:

  • max_features: None, 'auto', 'log2'
  • criterion: 'gini', 'entropy'
  • min_samples_split: 2, 3, 4, 5, 10

The grid search function tested all combinations of these parameters, and discovered that the following had the highest performance. These settings were then used in the final classifier.

  • max_features: 'auto'
  • criterion: 'gini'

Validation involves randomly splitting the dataset into separate training and testing groups. The model is trained on the training dataset, and then performance is evaluated on the test dataset. This process helps validate the performance on an independent dataset and guards against overfitting. For this analysis, due to the low number of POI, I used the statified shuffle split function in conjunction with grid search in scikit-learn for cross validation over 1000 folds. I also used the test_classifier method from the tester script provided, which also performs stratified cross-valudation evaluation.

Two metrics that measure model performance are precision and recall. Precision measures the ratio of true positives to the total true positive and false positives. In other words, correctly identifying POI provided that the person is actually a POI. Recall measues the ratio of true positives to the total true positives and false negatives. In other words, the chances that the algorithm predicts a POI when in fact the person is a POI.
